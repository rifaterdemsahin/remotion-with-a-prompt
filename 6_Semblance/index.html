🐛 Known Issues & Fixes
⚠️ fetch() fails on file:// protocol Warning
When opening index.html directly (file:// URL), fetch() calls to load menu JSON files fail with CORS errors.
Fix: Use npx serve . or a local HTTP server. GitHub Pages serves over HTTPS so this doesn't affect production.
✅ Canvas preview doesn't match Remotion output Resolved
The Canvas 2D API preview is an approximation. Spring physics simulation differs from Remotion's exact spring() implementation.
Fix: Added "CSS Preview (live render)" label to set expectations. The code output is the ground truth.
✅ Cross-origin images blocked in Canvas Resolved
When loading external images into Canvas for preview, the browser blocks it if the image host doesn't send CORS headers.
Fix: Added crossOrigin="anonymous" to Image elements. Base64 data URIs always work. URL images may still fail on restricted hosts.
🔴 Complex multi-object prompts not fully parsed Open
Prompts with multiple sequential animations (e.g., "first fade in, then slide, then zoom out") only apply the last detected animation type.
Workaround: Use separate Sequence components. Future work: add timeline/sequence support to the parser.
⚠️ Nav JSON fetch shows 404 in subfolder pages on file:// Warning
Subfolder pages use data-root="../" to find JSON menu files, but this path resolution fails on file:// URLs.
Fix: Use npx serve . from repo root. Confirmed working on GitHub Pages.
